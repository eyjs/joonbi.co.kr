# /code-review - 코드 리뷰

코드 품질을 검토하고 개선점을 제안합니다.

## 검토 항목

### 1. 코드 스타일
- [ ] 네이밍 컨벤션 준수
- [ ] 파일 구조 적절성
- [ ] 주석 품질

### 2. 타입 안전성
- [ ] any 타입 사용 여부
- [ ] 명시적 return type
- [ ] null/undefined 처리

### 3. 에러 처리
- [ ] try/catch 적절성
- [ ] 에러 메시지 명확성
- [ ] 에러 로깅

### 4. 보안
- [ ] 입력 검증
- [ ] SQL Injection 방지
- [ ] 인증/인가 체크

### 5. 성능
- [ ] N+1 쿼리 문제
- [ ] 불필요한 연산
- [ ] 메모리 누수 가능성

### 6. 테스트
- [ ] 테스트 커버리지
- [ ] Edge case 처리
- [ ] Mock 적절성

## 출력 형식

```markdown
## 📋 코드 리뷰 결과

### ✅ 잘된 점
- ...

### ⚠️ 개선 필요
| 심각도 | 파일 | 이슈 | 해결 방안 |
|--------|------|------|----------|
| 🔴 높음 | ... | ... | ... |
| 🟡 중간 | ... | ... | ... |
| 🟢 낮음 | ... | ... | ... |

### 📊 종합 평가
완성도: ⭐⭐⭐⭐ (80%)
```
